# Phase B - Cabin Foundation

Phase B introduces the first cabin-specific runtime modules for AURA DCOS.

## Packages

- surfaces
- hal
- digital-twin
- capabilities

## Surfaces

The surfaces package stores digital cabin surface records and tracks ID, type, priority, state and energy.

## HAL

The HAL package provides a common interface for prototype and future hardware devices.

## Digital Twin

The digital twin package stores a virtual cabin context including mode, speed, vehicle state, weather and occupants.

## Capabilities

The capabilities package provides a controlled extension model for future cabin features.

## Next Phase

Phase C should build the first runnable AURA Studio app that wires together the runtime packages.
