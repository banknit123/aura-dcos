# AURA Companion Ecosystem

The Companion Ecosystem turns AURA from a single assistant into a user-personalized companion platform.

## Package

`@aura-dcos/companion-ecosystem`

## Purpose

Users can choose or create their own in-cabin companion while AURA preserves brand quality, safety and driver-distraction limits.

## Companion Types

Supported companion categories include:

- Light beings
- Dogs
- Cats
- Owls
- Foxes
- Pandas
- Dolphins
- Dragons
- Phoenixes
- Robots
- Human avatars
- Custom companions

## Customization

A companion profile includes:

- Name
- Species
- Role
- Personality
- Appearance colors
- Size
- Render style
- Voice tone
- Preferred cabin surfaces
- Movement style
- Interaction frequency
- Marketplace readiness

## Movement and Safety

Companions can walk, fly, float, perch, sit, fade or move between display surfaces. The behavior policy changes based on vehicle motion state:

- Parked: full expression allowed.
- Low speed: reduced motion and no driver-visible distraction.
- Driving: subtle presence only, usually passenger/rear/ambient surfaces.

## Default Companion

AURA's default companion remains a premium light-being. Animal companions and custom avatars are optional user-personalized experiences.

## Production Notes

Future Studio screens can use this package to build a Companion Creator UI, companion marketplace and occupant-specific companion profiles.
